## Быстрый старт
```bash
git clone https://github.com/LidaMudota/task_Management_Web_App_-MVP-skeleton-
cd task_Management_Web_App_-MVP-skeleton-
npm install
npm start
```
По умолчанию dev-сервер поднимается на **http://localhost:3000** и автоматически открывает вкладку.

### Сборка production
```bash
# Для Node 17+ (включая Node 20) добавьте совместимость с OpenSSL:
NODE_OPTIONS=--openssl-legacy-provider npm run build
```
Результат сборки появится в каталоге `public/` (игнорируйте или очистите его перед коммитом, если не выкатываете статический билд).

## Переменные окружения
Проект работает без внешних сервисов и не требует `.env`. Все данные (пользователи, задачи) сохраняются в `localStorage` текущего браузера.

## 3D-фон и доступность
- В шапке и фоне добавлен слой **depth-weave**: параллакс-линиями на canvas с мягкими свечениями. Он использует `pointer-events: none`, поэтому не мешает кликам.
- Качество автоматически подстраивается под `devicePixelRatio`, чтобы не перегружать слабые устройства.
- Если в системе включён `prefers-reduced-motion`, canvas отключается, а остаются только статичные мягкие подсветки.

## Troubleshooting
- **Node 17+ / OpenSSL error** — при `npm run build` добавьте `NODE_OPTIONS=--openssl-legacy-provider`. Это же можно сделать временно перед командой или в окружении CI.
- **Порт 3000 занят** — остановите процесс, использующий порт (`lsof -i :3000`), либо запустите dev-сервер с флагом `--port <номер>`: `npm start -- --port 4000` (webpack-dev-server примет переопределение).
- **Белый экран/старые данные** — очистите `localStorage` домена (данные пользователей/задач хранятся там) и перезагрузите страницу.
- **Не открывается вкладка автоматически** — зайдите вручную на `http://localhost:3000` (авто-открытие может блокироваться настройками системы).
